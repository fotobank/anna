// Настройка Inter_Error::$conf :
/ **
* Ключи и значение по умолчанию в статическое свойство Inter_Error::$conf :
* DebugMode:
* Открыть режим отладки? Если это так, он будет показывать в конце веб-страницы в веб-браузере
* Опция: ложь (по умолчанию) / правда
* Переменная типа: BOOL
*
* FriendlyExceptionPage:
* В то время как DebugMode ложно и происходят исключение, которое HTML файл должен быть дисплей (с помощью 'требуют')
* Оставьте это поле пустым, если вы не хотите, чтобы вступили в силу.
* Опция:option: ''(blank, default)/любое значение каталог
* Переменная типа: строка
*
* logType:
* Определить, как войти ошибок.
* 'detail': войти с информацией трассировки. Не рекомендуется использовать это значение в развертывания среды, если вы не сталкиваются с трудной проблемой
* 'simple': войти без информации трассировки.
* false: do not log
* option: false(default)/'detail'/'simple'
* Переменная типа: BOOL | строка
*
* LogDir:
* Каталог для сохранения файла журнала. Не имеют '/' или '\' в конце сценария
* На эту настройку влияет "logType". Если 'logType' является ложным, он ничего не будет делать.
* Но если 'logType' не является ложным, и этот параметр пуст или не является каталогом. он будет will ошибку с помощью настройки в php.ini.
* option: ''(blank, default)/any directory value
* var type: string
*
* suffix:
* Суффикс файла журнала
* Опция: '-Inter-ErrorLog.log' (по умолчанию)
* Переменная типа: var type: string
*
* variables:
* Ключ, который вы хотите экспортировать и показать переменные в $ GLOBALS
* Опция: array ("_GET", "_POST", "_SESSION", "_COOKIE") (по умолчанию)
* Переменная типа: array
*
* ignoreERROR:
* Некоторые программы могут легко генерировать много E_NOTICEs и E_STRICTs. Можно опустить их, изменив настройки массива 'IgnoreError'
* Предопределенные константы в Error Handling: http://docs.php.net/manual/en/errorfunc.constants.php
* По умолчанию, он не будет войти E_STRICT. Но вы должны быть осторожны, чтобы не перезаписать настройки по умолчанию.
* Вот пример:
* <pre>
* Inter_Error::$conf['ignoreERROR'] = array(E_NOTICE, E_STRICT); / / перепишет настройки ignoreERROR умолчанию, но остаются не войти E_STRICT
* </pre>
* option: array(E_STRICT)(default)
* var type: array
* /
